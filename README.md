## 文档 Schema（`docs/`）

以下约定适用于 **PRD 及同类文档**；**唯一规范目录为 `docs/`**（不使用名为 `specs` 的子目录作为约定位置）。

### 放置与格式

| 规则 | 说明 |
|------|------|
| **位置** | 文件必须放在 `docs/` 下（与 [`docs/README.md`](./docs/README.md) 同级，或 `docs/` 下的子目录内）。**不要**再使用 `docs/superpowers/specs/` 或仓库内其他「规格专用」路径作为约定。 |
| **格式** | 默认 Markdown（`.md`）。若使用其他格式，仍须在 [`docs/README.md`](./docs/README.md) 的索引表中登记。 |
| **命名建议** | `YYYY-MM-DD-<主题短横线-slug>.md`，便于排序与检索（例：`2026-04-05-boss-direct-review-design.md`）。 |

### 索引维护（强制）

每 **新增、迁入或重命名** 一个 PRD 类文件，**必须** 同步更新 [`docs/README.md`](./docs/README.md) 中的 **「文档索引」** 表。

索引中每个文件对应一行，并写清：

1. **产物 / 来源** — 文档如何产生（例：评审定稿、会议纪要整理、与某需求对应的规格）。
2. **概要** — 主要内容与范围（关键决策、分阶段结论等）。
3. **用途** — 读者与目的（例：指导排期、评审依据、防止需求漂移）。
