## 1. 设计原则

### 1.1 视觉风格

参考 Boss 直聘招聘方（HR）后台界面的视觉语言，建立与之产生视觉关联的设计体系，强化产品的「职场社区」定位与模拟游玩的代入感。

核心原则：
1. **视觉关联而非抄袭**：借鉴 Boss 直聘的色彩与信息密度，但保持独立产品身份，不使用其 Logo 或官方元素。
2. **数据优先**：榜单与统计数字是主角，设计服务于数据可读性。
3. **克制的梗文化**：界面保持专业感，梗向内容通过文案和运营皮肤承载，不通过视觉噪音表达。
4. **透明可信**：每处数据来源、模拟说明、免责声明都有清晰的 UI 承载位置，不藏不躲。

### 1.2 平台范围

- **主场景**：PC 浏览器（1280px 及以上宽度）
- **兼容场景**：移动浏览器（375px 起），布局响应式折叠，核心功能可用
- **不包含**：原生 App（iOS/Android）

## 4. 页面规格

### 4.1 首页 / 榜单列表页

#### 布局

```
┌────────────────────────────────────────────┐
│  榜单 Tab 切换栏                              │
│  [已读不回 HR] [简历黑洞公司] [Nice HR] [...]   │
├──────────────────────┬─────────────────────┤
│                      │                     │
│   榜单主列表（左栏）     │  侧边栏（右栏）         │
│   ~640px             │  ~280px             │
│                      │                     │
└──────────────────────┴─────────────────────┘
```

#### 榜单 Tab 切换栏

- 水平滚动 Tab，移动端支持左右滑动
- 当前 Tab：`color-primary` 文字 + 底部下划线
- Tab 间距：`spacing-md`

#### 榜单列表条目

每条条目为卡片形式，包含：

```
┌──────────────────────────────────────────────┐
│  #1  张三 · 某大厂科技                  [负向标签] │
│       招聘岗位：产品经理                           │
│       提名数：128  最近更新：3天前                  │
│                              [提名 / 查看详情]  │
└──────────────────────────────────────────────┘
```

- 排名数字：`font-size-h2`，`color-primary`（前3名），`color-neutral`（其余）
- 负向标签（如「已读不回」）：`color-negative` 背景浅红 `color-negative-light`，小圆角 Tag
- 正向标签（如「Nice HR」）：`color-primary-light` 背景
- 低样本状态（样本建设中）：灰色蒙层 + 「数据建设中，敬请期待」占位文字，不显示排名数字
- 条目间距：`spacing-sm` 间隔线

#### 侧边栏

- **提名入口卡片**：醒目的「提名一位 HR」CTA，`color-primary` 按钮
- **榜单说明**：一句话解释当前榜单排序口径（「综合提名数量与近期更新」）
- **数据简报预告**（P1）：本周简报入口

### 4.2 HR 详情页

```
┌──────────────────────────────────────────────┐
│  ← 返回榜单                                    │
├──────────────────────────────────────────────┤
│  HR 展示名  ·  公司名                           │
│  招聘岗位                                      │
│  [已读不回 ×86] [Nice ×12]  提名总数：98          │
│  最近提名：2天前                                 │
├──────────────────────────────────────────────┤
│  [练一局 →]   [提名 / 更新提名]                   │
├──────────────────────────────────────────────┤
│  提名记录列表（脱敏，不显示提名人身份）               │
│  每条：招呼文本摘要（截断） + 院校层级 · 专业门类      │

```

- 「练一局」按钮：`color-primary` 实心按钮，跳转猜 Boss 并预填 `hr_id`
- 提名记录列表：仅展示招呼摘要（前30字截断）+ 粗粒度背景标签，不显示提名用户信息

### 4.3 提名写入页

单栏居中表单（最大宽度 `640px`）：

| 字段 | 组件 | 说明 |
|------|------|------|
| HR 姓名 | 文本输入 | 实时显示规范化预览（如「张三（HR）」→「张三」） |
| 公司名 | 文本输入 + 下拉候选 | 输入时联想高频公司名单 |
| 招聘岗位 | 文本输入 | 必填 |
| 职位描述（JD） | 多行文本 | 可选，用于猜 Boss 局内 Boss 卡展示 |
| 标签 | 多选 Tag | 已读不回 / 沉默 / Nice / 秒回 / 其他（可扩展） |

- 提交按钮：`color-primary`，宽度 100%
- 一人一票提示：表单顶部橙色 Banner「你对该 HR 已有提名记录，此次提交将覆盖原有内容」（已有提名时显示）
- 登录门禁：未登录点击提名 → 弹出登录弹窗，登录后回填表单

### 4.4 猜 Boss 落地页（P1）

#### 入口双卡片

```
┌─────────────────────┐  ┌─────────────────────┐
│   🎮 模拟数据          │  │   👥 真实数据          │
│                     │  │                     │
│  系统生成的练习素材    │  │  来自用户真实提名      │
│  适合新手练习         │  │  需已有提名记录        │
│                     │  │                     │
│   [开始练习]         │  │   [进入真实局]        │
└─────────────────────┘  └─────────────────────┘
```

- 两卡片等宽并排，`radius-lg`，`shadow-card`
- 模拟数据卡：左侧，默认推荐（P0 冷启动期视觉权重略高）
- 真实数据卡：右侧，数据不足时按钮灰色 + Tooltip「当前真实素材较少，建议选择模拟数据」
- 模拟数据标识：橙色 Badge「练习素材」，`color-synthetic-badge`

### 4.5 猜 Boss 游玩页（P1）

三栏布局，模拟 Boss 直聘招聘方后台视觉风格：

```
┌─────────────┬──────────────────────┬──────────────┐
│             │   顶栏：当前求职者信息   │              │
│  左栏        ├──────────────────────┤   右栏        │
│  未读招呼列表  │                      │   Boss 卡     │
│  （K条）     │   主消息区             │              │
│             │   招呼正文             │  HR 展示名    │
│  每条显示:    │   院校层级 · 专业门类   │  公司名       │
│  - 摘要      │                      │  岗位名       │
│  - 标签      │   [会回复] [不回复]    │  JD 正文      │
│  - 未读圆点   │                      │              │
└─────────────┴──────────────────────┴──────────────┘
```

**列宽比例**：左栏 `220px` / 主栏 `flex: 1` / 右栏 `260px`

**左栏（未读列表）**
- 背景：`#F0F0F0`（模拟 IM 侧栏）
- 当前回合条目：白色背景 + 左侧 3px `color-primary` 竖线
- 已答条目：灰色文字，已回复/不回复图标标记
- 未读圆点：`color-negative` 红点

**顶栏（求职者信息区）**
- 高度：52px，白色背景，底部分割线
- 内容：`院校层级` Tag + `专业门类` Tag + 「第 X / K 轮」进度提示
- 数据来源标记：若为模拟数据，右侧橙色小 Badge「练习素材」

**主消息区**
- 招呼文本：气泡样式，左对齐（模拟对方发来消息），最大宽度 70%，`radius-md`，浅灰背景
- 操作按钮：固定在消息区底部，两个等宽按钮
  - 「会回复」：`color-primary` 实心
  - 「不回复」：白色 + `color-border` 描边
- 已答回合：操作按钮区域替换为「已选择：XX」灰色提示，不可修改

**右栏（Boss 卡）**
- 白色背景，`radius-md`，`shadow-card`
- HR 展示名：`font-size-h3` `font-weight-bold`
- 公司名 + 岗位：`font-size-body` 次要色
- JD 区域：`font-size-small` 可滚动，最大高度 `200px`，溢出渐隐 + 「展开」

**进度条**
- 顶部导航栏下方全宽绿色进度条，`color-primary`，`height: 3px`

### 4.7 设置页 / 求职者档案

单栏居中（最大宽度 `560px`）：

| 字段 | 组件 | 说明 |
|------|------|------|

| 专业门类 | 单选按钮组 | 文科 / 商科 / 理科 / 工科 / 交叉·其他 |

- 保存按钮：`color-primary` 实心，宽度 100%
- 档案说明：表单顶部灰色说明文字「你的招呼语和背景信息将用于为其他用户提供真实练习素材，不会展示你的真实身份」

### 4.8 分享卡片（P1）

- 生成方式：服务端渲染图片（OG 预览）或端上 Canvas 导出
- 尺寸：`1080×1080px`（正方形，兼容小红书、微博）
- 构成：
  - 背景：`color-primary` 渐变或深色底
  - 核心数字：最大字号（如「128人 已读不回」）
  - 产品品牌：「Boss直评」Logo + 小字「社区统计，仅供参考」
  - 免责声明：底部小字「数据来自用户提名，非官方数据」
- 不默认显示具体 HR 姓名（用公司级/行业级结论）

## 5. 组件规范

### 5.1 按钮

| 类型 | 背景 | 文字色 | 边框 | 用途 |
|------|------|--------|------|------|

| Default | 白色 | `color-text-primary` | `color-border` | 次要操作 |
| Danger | `color-negative` | 白色 | 无 | 危险操作 |

- 高度：`36px`（默认）/ `44px`（大号，移动端主操作）
- 圆角：`radius-sm`
- Hover：亮度 +5%；Active：亮度 -5%

### 5.2 Tag / Badge

| 类型 | 背景 | 文字色 | 用途 |
|------|------|--------|------|
| 负向 | `color-negative-light` | `color-negative` | 已读不回、黑洞公司 |
| 正向 | `color-primary-light` | `color-primary` | Nice HR、秒回侠 |
| 模拟标识 | `#FFF7E6` | `color-synthetic-badge` | 模拟数据说明 |
| 中性 | `#F5F5F5` | `color-text-secondary` | 院校层级、专业门类 |

- 高度：`22px`，`radius-sm`，`padding: 0 8px`，`font-size-small`

### 5.3 卡片

- 背景：`color-bg-card`（白色）
- 圆角：`radius-md`
- 阴影：`shadow-card`

### 5.4 空状态

- 榜单空页：居中插图（可选）+ 「暂无数据，成为第一个提名的人」+ 「去提名」按钮

## 6. 交互规范

### 6.1 导航与路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 首页 / 榜单列表 | 默认展示「已读不回 HR 榜」 |
| `/board/:type` | 指定榜单 | `type` 为榜单类型枚举 |
| `/hr/:id` | HR 详情页 | — |
| `/nominate` | 提名写入页 | 需登录 |
| `/play` | 猜 Boss 落地页（P1） | — |
| `/play/session/:id` | 游玩页（P1） | — |
| `/play/result/:id` | 结算页（P1） | — |
| `/settings` | 设置页 | 需登录 |
| `/weekly` | 每周简报（P1） | — |

### 6.2 登录门禁

- 写入类操作（提名、档案保存）必须登录
- 未登录触发写入 → 弹出登录弹窗 → 登录后回到原操作
- 浏览类操作（榜单、HR 详情、游玩）无需登录

### 6.3 加载态

- 页面级加载：顶部 3px 进度条（`color-primary`）
- 列表加载：骨架屏（Skeleton），与实际条目等高
- 按钮提交中：按钮内 Spinner + 禁用点击，避免重复提交

### 6.4 Toast 与反馈

| 场景 | 类型 | 文案示例 |
|------|------|---------|
| 提名提交成功 | 成功（绿） | 「提名已提交」 |
| 提名更新成功 | 成功（绿） | 「提名已更新」 |
| 表单校验失败 | 错误（红） | 「请填写 HR 姓名」 |
| 网络错误 | 错误（红） | 「提交失败，请稍后重试」 |
| 档案保存成功 | 成功（绿） | 「档案已保存」 |

- Toast 位置：顶部居中，`z-index` 最高层
- 自动消失：3 秒

### 6.5 游玩页交互细节（P1）

- 每轮操作后：主操作按钮替换为「已选择」灰色提示，200ms 过渡动画后自动进入下一轮
- 最后一轮答完：延迟 500ms 后跳转结算页，避免过于突兀
- 离开游玩页确认：若会话进行中，弹出确认弹窗「离开将记录为未完成局，确定离开？」
- 左侧列表点击：可跳回已答回合查看，但不可修改

### 6.6 响应式断点

| 断点 | 宽度 | 布局变化 |
|------|------|---------|
| Desktop | ≥1280px | 标准三栏 / 双栏布局 |
| Tablet | 768px–1279px | 侧边栏折叠为抽屉，主内容区全宽 |
| Mobile | <768px | 单栏，顶部导航折叠为汉堡菜单，猜 Boss 页左侧列表改为顶部进度指示器 |
