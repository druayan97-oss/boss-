# 猜 Boss 行为 — 功能与交互清单

**说明**：本文档只描述 **「猜 Boss 行为」** 这一模块的 **功能与交互**，不涉及后端实现。完整产品定义见 [2026-04-05-boss-direct-review-design.md](./2026-04-05-boss-direct-review-design.md)。

---

## 一、这是什么产品（简要）

**Boss直评**面向求职者：大家可以在平台上记录、讨论招聘方（以 HR 为主）的求职体验（例如是否已读不回等），并形成榜单与话题，让求职信息更透明一些。

其中 **「猜 Boss 行为」** 是一个 **练习型玩法**：你 **假装自己是某位真实存在于本平台榜单里的 HR**，会陆续收到一条条 **求职招呼**（带简单的学历档位、专业门类标签）。你要判断：**如果你是这位 HR，你会回复这条消息，还是不回复？** 一局结束后，系统会告诉你猜对了多少轮，并说明标准答案大致依据什么规则生成——**仅供娱乐与练习，不代表 Boss 直聘官方，也不等于真实聊天记录。**

## 二、功能与交互清单（猜 Boss 行为）

### 2.1 从哪里进入

- [ ] 底部或主导航中有 **「模拟」**（或同等名称）入口，进入猜 Boss 玩法。
- [ ] 在 **榜单 / 某位 HR 的详情页** 可点击 **「代入 Ta 练一局」**，直接进入玩法并 **预选这位 Boss**。

### 2.2 设置页：我的求职画像（与提名共用）

- [ ] 用户可在 **设置** 里维护：
  - **招呼语**：自己平时给 Boss 发消息用的自我介绍/开场模板。
  - **院校层级**（单选，粗粒度）：如清北/Top3、C9、211、双非一本、二本、专科/高职、海外（粗）、不便透露等。
  - **专业门类**（单选）：文科、商科、理科、工科、交叉/其他等。
- [ ] **不要求填写具体学校全称**，避免过度暴露隐私。
- [ ] 提示文案可说明：**当你提名某位 HR 时，会带上这里的招呼语和标签，供其他玩家在「真实池」里练习**（若尚未提名，可引导去提名）。

### 2.3 与「提名」的配合（真实招呼从哪来）

- [ ] 用户 **提名某位 HR** 时，使用 **当前设置页里的招呼语 + 院校层级 + 专业门类**，作为「我向这位 Boss 打招呼时用的那一套」。
- [ ] **同一位 Boss 每人只有一条有效提名**；若用户 **改设置后再保存/更新提名**，其他人之后在真实池里看到的是 **更新后的版本**。
- [ ] 其他玩家在玩 **真实池**、且代入你提名过的那位 Boss 时，可能刷到你贡献的这条招呼（**不显示你的昵称或任何身份信息**）。

### 2.4 开局前：选「真实」还是「练习」

- [ ] 开一局前，用户选择 **数据来源**（如两个 Tab 或开关）：
  - **真实池**：招呼来自 **真的提名过这位 Boss 的用户**（见上节）；若条数不够，用平台准备的 **示例文案** 凑满一局，并在结算或说明里提示 **含示例内容**。
  - **练习池**：招呼为 **系统用模板拼出来的练习内容**，界面上有 **明显提示**（如「本局为练习素材，非真实用户提名」）。
- [ ] **同一局内只使用一种来源**，不混用。
- [ ] 默认可优先 **练习池**，避免冷启动时真实条数太少导致无法开局。

### 2.5 选要代入的 Boss

- [ ] **自己选**：从列表里选一位 Boss（列表来自本平台已有榜单数据；若该 Boss 数据不足，提示换人或改用随机）。
- [ ] **随机一位**：系统随机分配一位 **数据量达标** 的 Boss；若暂无可选，提示稍后再试或改用练习池等。

### 2.6 局内界面（像收消息一样）

- [ ] 一局包含 **固定多轮**（例如十几轮，具体产品可定）。
- [ ] 每一轮展示：
  - **招呼正文**；
  - **小标签**：院校层级 · 专业门类（与设置里枚举一致）。
- [ ] **真实池**下不展示发送者是谁；**练习池**下保持同样版式，仅靠文案区分来源。
- [ ] 每轮两个主操作：**「会回复」**、**「不回复」**（二选一）。
- [ ] **首轮建议不在每一轮立刻告诉用户对错**，避免被轻易摸透规律；**一局结束再在结算页统一展示**。（后续若产品要做「每轮即时反馈」，可作为增强项。）

### 2.7 结算页

- [ ] 展示本局 **猜对比例 / 准确率**、**总轮数**。
- [ ] 标明本局是 **真实池** 还是 **练习池**。
- [ ] 展示一段 **固定说明文案**：标准答案大致依据 **该 Boss 在本平台的标签/统计** 模拟生成，**仅供练习与讨论，不代表对方真实会这样回复**。
- [ ] 可展示与本局相关的 **一个概括性数字**（例如「不回」倾向的粗略概率），用语通俗，避免技术术语。
- [ ] 操作：**再来一局**、**换一位 Boss**、**返回榜单**。

### 2.8 体验与限制（用户能感知到的）

- [ ] 输入过长或明显违规的招呼，在设置或提名时 **提示修改** 或 **无法保存**（具体话术产品定）。
- [ ] 必要时 **限制每日可玩局数**，触限时给出 **Toast / 弹窗说明**。

### 2.9 后续可增强的交互（可选，不阻塞首版）

- [ ] 在真实池某条招呼旁提供 **举报** 入口（选理由、简短说明），处理结果用户侧表现为 **内容下架** 等（流程与全局举报策略一致即可，此处不展开实现）。
- [ ] **个人战绩**：历史局数、准确率趋势、按 Boss 维度的简单统计。
- [ ] **开关**：是否 **每轮结束立即显示对错**。
- [ ] **开关或玩法**：单局内 **混合** 真实招呼与练习招呼（若产品上需要）。
- [ ] **轻量排行榜**（如练习达人），若做需配套说明与防刷体验。

---

## 三、对用户说明的边界（文案层面）

- [ ] 全链路避免让人以为 **与 Boss 直聘官方实时同步** 或 **在回放真实微信**。
- [ ] **练习池** 必须与 **真实用户提名** 在文案上区分清楚。
- [ ] **真实池** 仅展示 **档位与门类 + 招呼正文**，不出现 **具体学校全名、电话** 等敏感信息。
