# 猜 HR / Boss 行为 — 功能清单

**关联主规格**：[2026-04-05-boss-direct-review-design.md](./2026-04-05-boss-direct-review-design.md) §3.3、§3.3.12  
**说明**：本清单按 **P1 / P2** 拆分，便于排期与验收；细节参数以主规格为准。

---

## 一、产品目标（验收口径）

- 用户 **代入打榜中真实存在的 HR**（`hr_id`），以 **Boss 视角** 对每条「求职招呼」判断 **会回复 / 不回复**（二态）。
- 系统根据该 HR 的 **社区提名统计** 生成 **可解释的标准答案序列**，计算 **准确率**；**非** Boss 直聘官方、**非**真实聊天记录回放。
- **P1** 交付 **真实素材池 + 模拟练习池** 两套数据源，用户 **开局前可切换**；**模拟数据生成器** 属 **P1**。
- **P2** 补齐 **举报、邮箱验证** 及 **体验与风控增强**（不推翻 P1 表结构）。

---

## 二、P1 功能清单

### 2.1 入口与导航

- [ ] 主导航 **「模拟」** Tab 进入玩法。
- [ ] 从 **榜单 / HR 详情** **「代入 Ta 练一局」** 深链进入，预填 `hr_id`。

### 2.2 设置页 · 求职者档案（与提名共用）

- [ ] 编辑并保存 **招呼语**（Boss 打招呼模板）。
- [ ] **院校层级** 单选枚举：清北/Top3、C9、211、双非一本、二本、专科/高职、海外（粗）、不便透露/其他（文案可配置）。
- [ ] **专业门类** 单选枚举：文科、商科、理科、工科、交叉/其他。
- [ ] **不写具体学校全名**；档案变更后，在 **下次保存提名** 时刷新对应 **`hr_greeting_candidate`**（与 §2.5 一人一票一致）。

### 2.3 提名联动（真实池素材）

- [ ] 用户 **提名某 HR** 时，用 **当前档案** 生成/更新 **`hr_greeting_candidate`**（`hr_id` + 招呼与标签快照）。
- [ ] 提名 **隐藏/失效**（含 P2 举报成立后）对应素材 **不可再被抽取**。

### 2.4 开局前 · 数据源切换

- [ ] 用户选择 **`data_source_mode`**：**真实提名池** / **模拟练习池**（Tab 或单选）。
- [ ] 默认可配置为 **模拟池**（冷启动友好）。
- [ ] **单局不混用** 两轨（真实与模拟不同时出现在同一局内）。

### 2.5 Boss 选择

- [ ] **指定 Boss**：从可玩列表选择（数据来自打榜 `hr_id` + 最低提名量门槛）。
- [ ] **随机 Boss**：在满足门槛的 `hr_id` 集合中均匀抽样；无可用项时提示重试或换模式。
- [ ] 随机/指定均需 **真实存在的 HR 实体**。

### 2.6 真实池 · 收件箱抽取

- [ ] 仅抽取 **`hr_id = 本局 Boss`** 且 `active` 的 **`hr_greeting_candidate`**。
- [ ] **排除当前玩家本人** 贡献的条目。
- [ ] 池内策略：**均匀随机** 或 **低曝光优先**（可配置）。
- [ ] 单局 **K 轮**；不足 K 条时用 **运营预置「真实风」模板** 补足，结算/脚注 **标明含示例**。
- [ ] 每轮 UI：**招呼正文** + **院校层级 · 专业门类** 标签；**不展示** 任何用户身份、昵称、具体校名。

### 2.7 模拟池 · 数据生成（P1 必做）

- [ ] **规则/模板生成** 招呼正文（片段拼接或整句模板表），**禁止 LLM**。
- [ ] **随机生成** `school_tier`、`field_track`（可配置分布）。
- [ ] 与真实池 **同一套** 长度上限、敏感词规则。
- [ ] 生成结果写入 **`simulation_turn` 快照**（或等价），并 **显著文案** 标明 **「练习素材，非真实用户提名」**。
- [ ] 仍代入 **真实 `hr_id`**；**标准答案** 仍基于该 HR **社区统计先验**（与招呼是否模拟无关）。

### 2.8 局内玩法

- [ ] 固定 **K 轮**（默认 12，可配置）；可选未来扩展限时模式（主规格可选）。
- [ ] 每轮二选一：**会回复** / **不回复**（与标准答案布尔字段对齐）。
- [ ] **首版不在每轮即时亮对错**（防反推随机规律）；结算页统一展示结果。

### 2.9 真值与结算

- [ ] 从提名聚合 `w_no_reply` / `w_positive` / `w_neutral`，**拉普拉斯平滑** + **`p_min`～`p_max` 截断** 得 `p_eff`，写入 **`simulation_session` 快照**。
- [ ] 用 **确定性随机种子** 对每轮伯努利抽样 **`ground_truth_is_no_reply`**，可复现、可审计。
- [ ] 结算展示：**准确率**、**K**、**本局数据源（真实/模拟）**、**固定模板解释文案**（含 `p_eff` 约值与非承诺说明）。
- [ ] 操作：**再来一局**、**换 Boss**、**回榜单**。

### 2.10 会话与状态

- [ ] 状态：`idle` / `boss_select` / `playing` / `completed` / `abandoned`。
- [ ] **弃局**（如 24h 未完成）标记与统计规则按主规格（可不计入胜率等）。

### 2.11 数据持久化（逻辑表）

- [ ] `job_seeker_profile`
- [ ] `hr_greeting_candidate`（按 `hr_id` 分池）
- [ ] `simulation_session`（含 `data_source_mode`、`p_no_reply_effective`、`rng_seed`、`k_rounds` 等）
- [ ] `simulation_turn`（招呼与标签快照、`ground_truth`、`user_chose`、`is_correct`）
- [ ] `hr_behavior_stats`（物化或视图，供 `p_eff` 计算）

### 2.12 API（可按端合并）

- [ ] `POST /simulation/sessions`（含 `hr_id` / `random` / `k` / `data_source_mode`）
- [ ] `GET /simulation/sessions/:id`
- [ ] `POST /simulation/sessions/:id/answers`（或等价）
- [ ] `GET /simulation/sessions/:id/summary`

### 2.13 P1 风控（基础）

- [ ] 招呼与模板 **长度限制**、**敏感词规则**。
- [ ] **基础频控**（如每用户每日局数上限，具体数值配置化）。

## 三、P2 功能清单

### 3.1 举报与真实 UGC 治理

- [ ] 用户侧 **举报入口**；对象含 **提名**、**`hr_greeting_candidate`（真实池）**。
- [ ] 核实成立后 **仅隐藏**（软隐藏、不参与榜与抽取）；不做首版自动 HR 降权/硬删（见主规格）。
- [ ] 模拟池模板问题以 **运营后台** 为主（用户举报可迭代）。

### 3.2 邮箱验证（全链）

- [ ] 与 §2.4 / §3.3.13 一致：验证邮件、`email_verified`、200 用户阈值与运营开关、未验证写入门禁等。

### 3.3 体验与风控增强（P2.2）

- [ ] 防刷加强：**每用户 / 每 Boss** 局数上限、异常脚本策略等。
- [ ] **历史曲线**、**按 Boss 维度** 战绩统计。
- [ ] 可选 **全站轻量排行**（低优先级，须防刷）。
- [ ] 可选 **单局混用真实/模拟回合**（若产品要开）。
- [ ] 可选 **即时每轮对错** 开关。
- [ ] 可选 **分层抽样真值**（固定 K·p 个「不回」再打乱）。
- [ ] 真实池 **重复检测**（如 SimHash）等。

---

## 四、边界与明确不做（摘要）

- 不连接 Boss 直聘等 **官方 API**；不宣称 **真实 HR 实时行为**。
- **真值**为 **启发式统计 + 随机抽样**，须在 UI 说明。
- **模拟数据** 不得冒充真实 UGC；**真实池** 不得展示可识别第三方隐私字段（具体校名、手机等）。

## 五、默认参数速查（可调）

| 项 | 建议默认 |
|----|----------|
| 单局回合数 K | 12 |
| 随机 Boss 最少提名量 N_min | 3（`n_total` 口径以实现锁定为准） |
| 拉普拉斯 α、β | 2、3 |
| p 截断区间 | [0.2, 0.8] |
| 弃局超时 T_abandon | 24h |
| 每用户每日局数上限（P1 基础） | 如 20（配置化） |
